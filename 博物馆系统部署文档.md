## 第一部分：项目概述

### 1.1 系统组成

| 项目 | 后端 | 前端 | 硬件 |
|------|------|------|------|
| 项目名称 | integrated-face-mqtt | 智慧博物馆管理界面 | 博物馆环境监控系统 (BearPi-HM_Nano) |
| 技术框架 | Spring Boot 2.7.18 | Vue3 + Vite + Tailwind CSS + ECharts | OpenHarmony + Hi3861V100 |

| 打包产物 | `integrated-face-mqtt-1.0.0.jar` | `dist/` 静态文件目录 | `.bin` 固件文件 |

### 2.3 部署环境要求

#### 2.3.1 硬件要求

| 项目 | 最低配置 | 推荐配置 |
|------|---------|---------|
| CPU | 4 核 | 8 核及以上 |
| 内存 | 8 GB | 16 GB 及以上 (AI 模型加载需要较大内存) |
| 磁盘 | 10 GB 可用空间 | 20 GB 及以上 (含模型文件) |
| 网络 | 可访问外网 (DeepSeek API、Tavily API) | 同左 |

> 说明：系统加载多个深度学习模型 (MTCNN、InsightFace、YOLOv8 等)，启动时内存占用较高，建议预留充足内存。

#### 2.3.2 软件环境

| 软件 | 版本要求 | 用途 | 安装说明 |
|------|---------|------|---------|
| JDK | 17 或以上 | 后端运行环境 | 推荐 Oracle JDK 17 或 OpenJDK 17 |
| Maven | 3.6 或以上 | 后端构建打包 | 也可使用项目自带的 Maven Wrapper (mvnw) |
| MySQL | 5.7 / 8.0 | 主业务数据库 | 需提前创建数据库和数据表 |
| MQTT Broker | Mosquitto 2.x 等 | IoT 消息中间件 | 需部署并确保端口可达 |
| Node.js | 18 或以上 | 前端构建与运行 | 下载 https://nodejs.org，安装时全部默认 |
| npm | 9 或以上 | 前端包管理 | 安装 Node.js 时自动包含 |

#### 2.3.3 外部 API 依赖

| 服务 | 用途 | 说明 |
|------|------|------|
| DeepSeek API | 大语言模型智能问答 | 需申请 API Key，通过 `https://api.deepseek.com` 调用 |
| Tavily API | 联网搜索能力 | 需申请 API Key，为智能助手提供实时搜索 |

> 部署机器需能访问外网以调用上述 API。

#### 2.3.4 AI 模型文件

系统依赖多个预训练 AI 模型，部署前需将模型文件放置到服务器指定目录。默认路径配置在 `application.yml` 中，可根据实际部署环境修改。

模型文件目录结构如下 (以默认路径为例)：

```
<模型根目录>/
├── FaceDetection/
│   └── mtcnn/                         # MTCNN 人脸检测模型 (目录，含多个模型文件)
├── LivenessDecetion/
│   └── anti/
│       └── IIC_Fl.onnx                # 活体检测 ONNX 模型 (单文件)
├── FaceRecognition/
│   └── InsightFace/
│       └── model_ir_se50.pt           # InsightFace 人脸识别 PyTorch 模型 (单文件)
├── fire/
│   └── best.onnx (或 best.pt)         # 火焰检测模型
└── abnormalBehavior/
    └── best.onnx (或 best.pt)         # 异常行为检测模型
```

对应 `application.yml` 中的配置项 (部署时按实际路径修改)：

```yaml
face:
  detect:
    model-path: <模型根目录>/FaceDetection/mtcnn
  liveness:
    model-path: <模型根目录>/LivenessDecetion/anti/IIC_Fl.onnx
  recognize:
    model-path: <模型根目录>/FaceRecognition/InsightFace/model_ir_se50.pt

vision:
  models:
    fire:
      path: <模型根目录>/fire
    crowd:
      path: <模型根目录>/person
    abnormal-behavior:
      path: <模型根目录>/abnormalBehavior
```

> 请将 `<模型根目录>` 替换为实际的模型存放路径，例如 `D:/SmartAImodel` 或 `/opt/models`。

### 2.4 构建打包

#### 2.4.1 获取源码

将项目源码拷贝或克隆到部署机器上：

```bash
# 如果使用 Git
git clone <仓库地址>
cd integrated-face-mqtt
```

#### 2.4.2 修改平台配置 (重要)

项目中 `pom.xml` 使用 JavaCV/OpenCV 进行图像处理，需要根据部署平台修改本地库的目标平台。

打开 `pom.xml`，找到 `<properties>` 中的 `javacv.platform` 属性：

```xml
<properties>
    ...
    <!-- 根据部署平台修改此值 -->
    <javacv.platform>macosx-arm64</javacv.platform>
    ...
</properties>
```

按部署环境修改为对应值：

| 部署平台 | 值 |
|---------|-----|
| Windows 64 位 | `windows-x86_64` |
| Linux 64 位 | `linux-x86_64` |
| macOS Intel | `macosx-x86_64` |
| macOS Apple Silicon | `macosx-arm64` (默认值) |

#### 2.4.3 执行 Maven 打包

```bash
# 方式一：使用系统安装的 Maven
mvn clean package -DskipTests

# 方式二：使用项目自带的 Maven Wrapper (无需安装 Maven)
# Linux / macOS
./mvnw clean package -DskipTests
# Windows
mvnw.cmd clean package -DskipTests
```

打包成功后，可执行 JAR 文件生成在：

```
target/integrated-face-mqtt-1.0.0.jar
```

> `-DskipTests` 跳过单元测试以加快打包速度。如需运行测试，去掉该参数即可。

#### 2.4.4 打包产物说明

| 文件 | 说明 |
|------|------|
| `target/integrated-face-mqtt-1.0.0.jar` | 可执行的 Spring Boot Fat JAR，内含所有依赖 |
| `face_data.db` | SQLite 本地人脸特征数据库 (运行时自动创建，无需手动准备) |

### 2.5 数据库配置

#### 2.5.1 安装 MySQL

确保目标服务器已安装 MySQL 5.7 或 8.0，并启动 MySQL 服务。

#### 2.5.2 创建数据库

登录 MySQL 后执行：

```sql
CREATE DATABASE museum DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
```

#### 2.5.3 创建数据表

本项目不包含自动建表功能 (MySQL 部分)，需手动执行以下 SQL 创建所有业务表：

```sql
USE museum;

-- 1. 设备表
CREATE TABLE museum_device (
    id            INT AUTO_INCREMENT PRIMARY KEY,
    device_id     VARCHAR(64) NOT NULL UNIQUE COMMENT '设备唯一标识',
    name          VARCHAR(128) COMMENT '设备名称',
    location      VARCHAR(256) COMMENT '安装位置',
    region        VARCHAR(128) COMMENT '所属区域',
    status        INT DEFAULT 0 COMMENT '在线状态：1-在线，0-离线',
    arm_status    INT DEFAULT 0 COMMENT '布防状态：1-布防，0-撤防',
    last_active   DATETIME COMMENT '最后活跃时间',
    create_time   DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    led_status    INT DEFAULT 0 COMMENT 'LED 状态',
    light_status  INT DEFAULT 0 COMMENT '灯光状态',
    buzzer_status INT DEFAULT 0 COMMENT '蜂鸣器状态',
    motor_status  INT DEFAULT 0 COMMENT '电机状态'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='博物馆设备表';

-- 2. 人员表
CREATE TABLE person (
    id          VARCHAR(64) PRIMARY KEY COMMENT '人员ID（UUID）',
    name        VARCHAR(128) NOT NULL COMMENT '姓名',
    role        VARCHAR(64) COMMENT '角色：visitor-访客，staff-工作人员',
    metadata    TEXT COMMENT '扩展信息（JSON 格式）',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='人员信息表';

-- 3. 门禁通行记录表
CREATE TABLE access_log (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    person_id   VARCHAR(64) COMMENT '人员ID',
    person_name VARCHAR(128) COMMENT '人员姓名',
    role        VARCHAR(64) COMMENT '角色',
    access_time DATETIME COMMENT '通行时间',
    granted     INT COMMENT '是否放行：1-通过，0-拒绝',
    device_id   VARCHAR(64) COMMENT '设备ID',
    reason      VARCHAR(512) COMMENT '原因说明'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='门禁通行记录表';

-- 4. 告警记录表
CREATE TABLE alarm_record (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    device_id   VARCHAR(64) COMMENT '设备ID',
    level       VARCHAR(32) COMMENT '告警级别：info/warning/critical',
    content     TEXT COMMENT '告警内容',
    confirmed   INT DEFAULT 0 COMMENT '是否确认：0-未确认，1-已确认',
    handled     INT DEFAULT 0 COMMENT '是否处理：0-未处理，1-已处理',
    handle_time DATETIME COMMENT '处理时间',
    handler     VARCHAR(128) COMMENT '处理人',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '告警时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='告警记录表';

-- 5. 告警规则表
CREATE TABLE alarm_rule (
    id             INT AUTO_INCREMENT PRIMARY KEY,
    name           VARCHAR(128) COMMENT '规则名称',
    condition_expr VARCHAR(512) COMMENT '条件表达式（Aviator 语法）',
    level          VARCHAR(32) COMMENT '触发告警级别：info/warning/critical',
    action_type    VARCHAR(64) COMMENT '动作类型：mqtt/websocket',
    action_target  VARCHAR(256) COMMENT '动作目标（主题名或 URL）',
    action_params  TEXT COMMENT '动作参数（JSON 格式）',
    enabled        INT DEFAULT 1 COMMENT '是否启用：1-启用，0-禁用'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='告警规则表';

-- 6. 工单表
CREATE TABLE work_order (
    id            BIGINT AUTO_INCREMENT PRIMARY KEY,
    alarm_id      BIGINT COMMENT '关联告警ID',
    title         VARCHAR(256) COMMENT '工单标题',
    description   TEXT COMMENT '工单描述',
    assignee      VARCHAR(128) COMMENT '指派人',
    status        VARCHAR(32) DEFAULT 'pending' COMMENT '状态：pending/processing/completed/closed',
    create_time   DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    complete_time DATETIME COMMENT '完成时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='工单表';

-- 7. 传感器数据表
CREATE TABLE sensor_data (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    device_id   VARCHAR(64) COMMENT '设备ID',
    smoke       INT COMMENT '烟雾值',
    temperature DECIMAL(10,2) COMMENT '温度（℃）',
    humidity    DECIMAL(10,2) COMMENT '湿度（%）',
    light       INT COMMENT '光照值',
    accel_x     DECIMAL(10,4) COMMENT '加速度 X 轴',
    accel_y     DECIMAL(10,4) COMMENT '加速度 Y 轴',
    accel_z     DECIMAL(10,4) COMMENT '加速度 Z 轴',
    pir         INT COMMENT 'PIR 人体红外：1-有人，0-无人',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '采集时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='传感器数据表';
```

#### 2.5.4 SQLite 人脸特征库

系统使用 SQLite 存储人脸特征向量，数据库文件 `face_data.db` 会在应用首次启动时自动创建 (位于 JAR 运行目录下)，无需手动初始化。

自动创建的表结构：

```sql
CREATE TABLE IF NOT EXISTS face_vector (
    id      TEXT PRIMARY KEY,    -- 人员ID
    name    TEXT NOT NULL,       -- 姓名
    feature TEXT NOT NULL        -- 人脸特征向量（JSON 序列化）
);
```

#### 2.5.5 修改数据库连接

### 2.6 外部服务配置

#### 2.6.1 MQTT Broker

##### 2.6.1.1 安装 MQTT Broker

推荐使用 Eclipse Mosquitto：

- Windows：从 https://mosquitto.org/download/ 下载安装
- Linux：`sudo apt install mosquitto mosquitto-clients`

安装后启动服务，默认监听端口 `1883`。

##### 2.6.1.2 修改 MQTT 配置

编辑 `application.yml` 中的 MQTT 配置，修改为实际的 Broker 地址：

```yaml
mqtt:
  host-url: tcp://<MQTT Broker 地址>:1883
  username:                    # 如 Broker 设置了认证，填写用户名
  password:                    # 如 Broker 设置了认证，填写密码
  clientId: spring-mqtt        # 客户端ID，同一 Broker 下需唯一
  timeout: 10                  # 连接超时（秒）
  keep-alive: 20               # 心跳间隔（秒）
  clean-session: true
  reconnect: true              # 断线自动重连
  qos: 1                       # 消息质量等级
  subscribe-topic: museum/+/sensors    # 订阅主题（接收传感器数据）
  send-topic: museum/+/actuators       # 发布主题（下发控制指令）
```

##### 2.6.1.3 MQTT 主题说明

| 主题模式 | 方向 | 说明 |
|---------|------|------|
| `museum/{deviceId}/sensors` | 设备 → 服务端 | 传感器上报数据 (JSON 格式) |
| `museum/{deviceId}/actuators` | 服务端 → 设备 | 控制指令下发 (JSON 格式) |

### 2.11 常见问题排查

#### Q1：启动报错 `java.io.FileNotFoundException: xxx模型路径`

原因：AI 模型文件未放置到 `application.yml` 配置的路径下。

解决：
1. 确认模型文件已拷贝到服务器
2. 检查 `application.yml` 中 `face.detect.model-path`、`face.liveness.model-path`、`face.recognize.model-path`、`vision.models.*` 的路径是否与实际一致
3. 注意路径分隔符：Windows 使用 `/` 或 `\\`，Linux 使用 `/`

#### Q2：MQTT 连接失败 `Connection refused`

原因：MQTT Broker 未启动或地址/端口不正确。

解决：
1. 确认 MQTT Broker (如 Mosquitto) 已启动：`systemctl status mosquitto`
2. 检查 `application.yml` 中 `mqtt.host-url` 的 IP 和端口
3. 检查防火墙是否放行 1883 端口
4. 如 Broker 设置了认证，确认 `mqtt.username` 和 `mqtt.password` 已填写

#### Q3：数据库连接失败 `Communications link failure`

原因：MySQL 服务未启动或连接信息错误。

解决：
1. 确认 MySQL 服务已启动：`systemctl status mysql`
2. 确认 `museum` 数据库已创建
3. 检查 `application.yml` 中数据库 URL、用户名、密码
4. 如 MySQL 在远程服务器，检查是否允许远程连接和防火墙规则

#### Q4：JavaCV 报错 `UnsatisfiedLinkError` 或 `NoClassDefFoundError`

原因：`pom.xml` 中 `javacv.platform` 与实际运行平台不匹配。

解决：
1. 打开 `pom.xml`，修改 `<javacv.platform>` 为当前平台值
2. 重新执行 `mvn clean package -DskipTests`
3. 用新生成的 JAR 替换部署的旧 JAR

#### Q5：DeepSeek API 调用失败 `401 Unauthorized`

原因：API Key 无效或未配置。

解决：
1. 确认 `AiConfig.java` 中的 `apiKey` 已替换为有效的 DeepSeek API Key
2. 确认 API Key 未过期且账户余额充足
3. 确认服务器可访问 `https://api.deepseek.com`

#### Q6：启动时内存不足 `OutOfMemoryError`

原因：AI 模型加载需要较大内存，默认 JVM 堆内存不足。

解决：启动时增大堆内存参数：
```bash
java -Xmx4g -jar integrated-face-mqtt-1.0.0.jar
```

#### Q7：端口被占用 `Address already in use: 8080`

解决：
```bash
# 查看占用端口的进程
# Linux
lsof -i :8080
# Windows
netstat -ano | findstr :8080

# 方式一：终止占用进程
# 方式二：更换端口启动
java -jar integrated-face-mqtt-1.0.0.jar --server.port=9090
```

#### Q8：前端页面空白 / CORS 错误

原因：直接双击打开了 `index.html`。

解决：必须通过 HTTP 服务器托管，不能用 `file://` 协议打开。运行 `npx http-server dist -p 8081` 后访问 `http://localhost:8081`。

#### Q9：摄像头无法使用

原因：浏览器要求 HTTPS 或 localhost 环境才能调用摄像头 (`getUserMedia`)。

解决：开发时使用 `localhost` 访问；生产环境必须配置 SSL 证书 (HTTPS)。

#### Q10：前端 JS 包体积过大

打包后约 2MB (gzip 后 552KB)，属于正常范围。如需优化可对 ECharts 按需引入组件。

| 执行器 | LED, 直流电机 (通风), 补光灯 |
| 自动控制 | 温度 >30°C 或湿度 >70% → 开启通风; 光照 <50lux → 开启补光灯 |
| MQTT Client ID | `bearpi_ia1` |
| 上报数据 | `{"temperature":,"humidity":,"light":,"timestamp":}` |

#### 4.2.2 E53_SC1 — 光照 (guang.c)

| 项目 | 说明 |
|------|------|
| 传感器 | BH1750 (光照, I2C 0x23) |
| 执行器 | LED, 补光灯 |
| 自动控制 | 光照 <20lux → 开启补光灯 |
| MQTT Client ID | `bearpi_light` |
| 上报数据 | `{"light":,"timestamp":}` |

#### 4.2.3 E53_IS1 — 人体红外 (ren.c)

| 项目 | 说明 |
|------|------|
| 传感器 | 人体红外感应探头 (GPIO_7 中断输入) |
| 执行器 | LED, 蜂鸣器 (PWM) |
| 功能特性 | 中断触发检测, 检测到人后蜂鸣器报警+LED 闪烁 300ms, 支持远程布防/撤防 |
| MQTT Client ID | `bearpi_is1` |
| 上报数据 | `{"human_detected":,"alarm_status":,"timestamp":}` |

#### 4.2.4 E53_SC2 — 加速度 (jiasud.c)

| 项目 | 说明 |
|------|------|
| 传感器 | MPU6050 (三轴加速度, I2C 0x68) |
| 执行器 | LED, LED_D1, LED_D2 |
| 功能特性 | 采集三轴加速度数据, 可用于震动/倾斜检测 |
| MQTT Client ID | `bearpi_accel` |
| 上报数据 | `{"accel_x":,"accel_y":,"accel_z":,"timestamp":}` |

#### 4.2.5 E53_SF1 — 烟雾 (yan.c)

| 项目 | 说明 |
|------|------|
| 传感器 | MQ-2 (烟雾/可燃气体, ADC 通道 6) |
| 执行器 | LED, 蜂鸣器 (PWM) |
| 自动控制 | 烟雾浓度 >100PPM → 蜂鸣器报警 + LED 闪烁 1s |
| MQTT Client ID | `bearpi_smoke` |
| 上报数据 | `{"smoke":,"timestamp":}` |

### 4.3 E53 接口引脚定义

BearPi-HM_Nano 的 E53 标准接口引脚分配如下 (根据代码中各驱动实际使用情况)：

| GPIO | 功能 | 适用模块 |
|------|------|---------|
| GPIO_0 | I2C1_SDA (数据线) | E53_IA1 / E53_SC1 / E53_SC2 |
| GPIO_1 | I2C1_SCL (时钟线) | E53_IA1 / E53_SC1 / E53_SC2 |
| GPIO_2 | 板载 LED | 所有模块 |
| GPIO_7 | 数字 I/O / 中断输入 | E53_SC1 (补光灯) / E53_IS1 (人体感应) / E53_SC2 (LED_D1) |
| GPIO_8 | 数字 I/O / PWM 输出 | E53_IA1 (电机) / E53_IS1 (蜂鸣器) / E53_SC2 (LED_D2) / E53_SF1 (蜂鸣器) |
| GPIO_14 | 补光灯控制 | E53_IA1 |
| ADC_CH6 | 模拟输入 | E53_SF1 (MQ-2 烟雾传感器) |

> 注意: 不同 E53 模块对同一 GPIO 引脚定义不同, 一次只能插一个扩展板。

### 4.6 快速开始

#### 4.6.1 环境要求

- OpenHarmony SDK (LiteOS-M 内核, 支持 Hi3861)
- BearPi-HM_Nano 开发板
- 一根 USB 串口线
- 串口调试工具 (HiBurn, 波特率 3000000)
- MQTT Broker 服务器

#### 4.6.2 配置 WiFi 和 MQTT

根据所选模块, 编辑对应 `.c` 文件中的 WiFi 和 MQTT 服务器信息:

```c
WifiConnect("YourSSID", "YourPassword");          // 修改 WiFi 名称和密码
NetworkConnect(&network, "192.168.1.100", 1883);  // 修改 MQTT 服务器 IP 和端口
```

#### 4.6.3 选择传感器模块

编辑 `T1_bwgjk/BUILD.gn`, 取消对应主程序和驱动文件的注释:

```gn
sources = [
    "wsg.c",        # E53_IA1 温湿度光照
    #"guang.c",     # E53_SC1 光照
    #"ren.c",       # E53_IS1 人体红外
    #"jiasud.c",    # E53_SC2 加速度
    #"yan.c",       # E53_SF1 烟雾
    "src/E53_IA1.c",
    #"src/E53_IS1.c",
    #"src/E53_SC1.c",
    #"src/E53_SC2.c",
    #"src/E53_SF1.c",
    "src/wifi_connect.c",
]
```

#### 4.6.4 编译

在 OpenHarmony 源码根目录执行:

```bash
python build.py BearPi-HM_Nano
```

#### 4.6.5 烧录

1. USB 串口线连接开发板到电脑
2. 打开 HiBurn, 选择对应 COM 口, 设置波特率 3000000，点击Connect
3. 按下开发板 RESET 键进入烧录模式
4. 选择编译产物 `.bin` 文件并烧录
5. 完成后点击HiBurn上的Disconnect，再次按下开发板 RESET键 重启
